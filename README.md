# **WeJump 투자 연구소 Data-Driven Investment Framework Repository**

이 곳([https://crowmag2.github.io/wejump/](https://crowmag2.github.io/wejump/)) 은 'WeJump 투자 연구소'가 데이터에 기반하여 독창적인 투자 시스템을 설계하고, 그 과정을 투명하게 기록하는 살아있는 문서 저장소입니다.

저희는 시장에 대한 막연한 '의견'이 아닌, 데이터로 검증된 '시스템'을 구축하고 이야기합니다.

### **핵심 원칙 (Core Principles)**

1.  **분석을 넘어 시스템으로 (Beyond Analysis, Towards Systems):**
    저희는 단편적인 시장 현상 분석에 그치지 않습니다. 아이디어 단계에서부터 백테스팅과 실 운용까지 고려한, 실제 **작동하는 '완성된 시스템'**을 설계하고 구현하는 것을 목표로 합니다.

2.  **데이터 기반의 투명성 (Data-Driven Transparency):**
    모든 전략과 주장은 데이터로 증명합니다. 이곳 GitHub 저장소는 그 아이디어와 고민, 검증의 과정을 가감 없이 공개하여 **결과에 대한 신뢰성**을 확보하는 공간입니다.

3.  **진화하는 프레임워크 (Evolving Frameworks):**
    궁극적으로 저희가 추구하는 것은 고정된 전략이 아닌, 스스로의 약점을 보완하고 시장에 적응하며 **끊임없이 진화하는 '투자 프레임워크'**입니다.

이 저장소의 문서와 데이터를 통해, 저희의 연구 여정을 확인하고 새로운 아이디어를 얻어 가시길 바랍니다. 더 깊이 있는 해설과 분석은 아래 채널에서 확인하실 수 있습니다.

# WeJump 투자전략 연구소 종합 역량 분석 보고서

## Executive Summary

WeJump 투자전략 연구소는 국내 **리테일 퀀트 생태계**에서 매우 독특한 위치를 차지하고 있습니다. 7개의 주요 AI 모델 분석을 종합한 결과, WeJump는 **"단일 코어 엔진(CBVR)을 실전 운용 가능한 시스템 수준까지 고도화한 개인 R&D 랩"**으로 정의됩니다.

**핵심 결론:**
- **리테일 퀀트 생태계**: 최상위 1~3% (투명성·시스템 설계 측면)
- **개인 퀀트 연구자 중**: 독보적 상위권 (운용 아키텍처 설계 관점)
- **공개 자료 기반 평가**: 국내 리테일 중 최고 수준의 투명성·재현성

**중요한 맥락:**
- 기관 퀀트팀과의 비교는 조직 규모·인프라·비공개 자료 등으로 직접 비교 불가
- WeJump는 **실전 사용을 목표로 한 개인 연구**로서, 전략 나열형 콘텐츠 채널과는 본질적으로 다른 접근
- 다전략 포트폴리오가 아닌 **단일 코어의 구조적 알파(Structured Alpha) 추구**가 핵심 철학

## I. 핵심 역량 분석

### 1. 전략 엔진 및 아키텍처 설계 (4.5/5.0)

#### 1-1. CBVR 3중 필터 시스템의 독창성

**개인 연구자 수준에서 보기 드문 체계적 설계:**

**Level 필터 (채널 기반 평균회귀)**
- EMA 기반 다층 Envelope 채널(±N%, ±2N%, ±4N%)
- 가격 위치에 따른 레버리지/인버스 비중 기계적 조절
- 평균회귀적 성격의 구간별 차등 비중 전략

**Speed 필터 (변동성 임계 트리거)**
- 변동성이 임계치(±6% 이상)를 넘을 때만 리밸런싱 발동
- **이벤트 기반(Event-driven) 실행**: 개인 투자자가 구현하기 어려운 고급 기법
- 실전 거래비용 최소화를 위한 설계 (단순 정기 리밸런싱 대비 우수)

**Trend 필터 (벡터 기반 추세 분석)**
- halfMA 기울기 등을 통한 수학적 벡터 분석
- 추세 방향과 강도의 객관적 수치화
- Level 필터 결정의 강화/약화 역할

#### 1-2. 재귀적 위험 관리 (Reflexivity) - 핵심 차별점

**Soros의 재귀성 이론을 개인 투자 시스템으로 구현:**
- 전략의 누적 수익 곡선(Portfolio Equity Curve) 자체를 입력으로 재사용
- 성과 곡선 꺾임 시 전역 디레버리징(Global Deleveraging) 자동 발동
- **리테일 퀀트에서는 거의 찾아볼 수 없는 메타 레벨 리스크 제어**

이는 단순 신호 생성이 아닌 **운용 제어 시스템(Controls)** 수준의 설계로, 기관에서나 볼 수 있는 문제의식을 개인이 구현한 사례입니다.

#### 1-3. 레짐 전환 메커니즘 - 실전 지향 설계

시장을 **평상(Normal) / 예외(Extreme)** 두 레짐으로 구분:
- Normal 국면: 평균회귀 모듈 중심 운용
- Extreme 국면: 추세순응·위기헤지 모듈 선별 적용
- **이중 확인(Double Confirmation)**: 내부 추세 붕괴 + 외부 VXX 폭발 동시 충족 시에만 헤지 발동

**개인 투자자 관점에서의 의미:**
- 불필요한 헤지 비용 억제 (실전 수익률 개선)
- 명확한 규칙 기반 실행 가능성
- "감으로 대응"이 아닌 시스템적 대응

### 2. 리스크 엔지니어링 및 자본 구조 설계 (4.5/5.0)

#### 2-1. 3-Bucket 자본 구조 (CBVR 2.3) - 기관형 사고의 개인 구현

**리테일에서는 보기 드문 자본 효율성 설계:**
- **Bucket 1**: 활성 증거금 (Active Margin)
- **Bucket 2**: MDD 범퍼 (Drawdown Buffer)
- **Bucket 3**: 복리 실물화 (Realized Compound & Safe Assets)

**실전적 의미:**
- 목표 MDD(예: -25%) 하에서 안전한 레버리지 운용
- 마진콜 리스크 체계적 관리
- 선물 운용의 구조적 비용을 버킷으로 흡수

**중요한 점:**
- CBVR 2.3은 **기관형 운용 아키텍처**로 설계됨
- 개인 투자자는 **CBVR 코어 신호로 ETF 비중 조절**하는 것이 현실적
- 2.3 방식은 운용 인프라와 리스크 관리 역량 필요

#### 2-2. 선물 복제 및 Decay 회피 - 구조적 알파 추구

**레버리지 ETF의 근본적 문제 인식:**
- ETF 비중 조절 → KOSPI200 선물(표준/미니)로 Target Beta 복제
- 레버리지 ETF의 **구조적 손실(Decay) 문제 회피**
- 증거금률(10~15%) 고려한 자본 효율화

**이는 단순한 전략 개선이 아닌:**
- 상품 구조 자체의 한계를 인식한 설계
- 구조적 알파(Structured Alpha)를 얻기 위한 금융공학적 접근
- 개인이 이 수준의 문제의식을 가진 것 자체가 특징적

#### 2-3. 헤지 모듈 진화 - 비용 최소화 집착

**CBVR 2.0 → 2.3 진화:**
- **2.0**: VXX 이중 확인(추세 붕괴 + 변동성 폭발)
- **2.3**: PFIX 동기화(금리 급등 대응 강화, Spitznagel 신호 동기화)

**실전 의미:**
- VXX/PFIX의 콘탱고/롤오버 비용 구조적 억제
- "언제/얼마나/어떻게" 헤지할지 명확한 규칙
- 헤지를 별도 포트폴리오가 아닌 **Extreme 레짐의 구성요소**로 통합

### 3. 검증 및 문서화 수준 (4.0~4.5/5.0)

#### 3-1. 강점: 장기 백테스트 및 논문형 정리

**2010~2025 구간 성과 (KODEX200/QQQ 4:6 합성)**
- CAGR: 32~35%
- MDD: -17~-20%
- Sharpe Ratio: 1.7~1.8
- Sortino Ratio: ~2.5

**백테스트 설계의 보수성 - 감점 요인이 아님:**
- **T-1 데이터 기반** (Look-ahead Bias 회피)
- **(시가+종가)/2 체결 가정**: 느슨하지만 현실적인 가격 반영
- **거래비용 과도하게 보수적 계산**: 실제로는 비중 조절만 발생하는데, 전체 자산 100% 매도 후 재매수로 가정
  - 이는 **안전마진을 둔 것**으로, 실제 성과는 백테스트보다 나을 가능성
- 모듈별 분리 제시 (코어 vs 헤지 vs 2.3 확장)

**개인 연구자로서는 매우 투명한 접근:**
- 가정을 명시하고 보수적으로 계산
- "좋아 보이게" 만들기보다 **실행 가능성** 우선

#### 3-2. 검증의 투명성 - 리테일 최상위 수준

**모듈별 효과 분리 제시:**
- 기본 엔진만의 성과
- VXX 헤지 추가 후 성과
- 2.3 확장 (선물+3-Bucket) 구조의 차이

**벤치마크 동시 제시:**
- KODEX200: CAGR 4.59%, MDD -40.65%
- QQQ(KRW): CAGR 19.33%, MDD -31.15%

**타 전략과의 구조적 비교:**
- 변동성 타겟팅, 리스크 패리티, 올웨더 등과 비교
- Pine Script 코드 기반 로직 step-by-step 설명

#### 3-3. 한계 인식 - 개인 연구의 현실적 제약

**자산군의 한계:**
- KOSPI/NASDAQ(+ 레버리지/인버스, VIX 계열) 중심
- 이는 개인이 접근 가능한 자산군의 현실적 제약
- 다전략 포트폴리오를 지향하지 않는 개인의 선택

**통계적 강건성 패키지:**
- OOS, 롤링 윈도우, 파라미터 민감도, 몬테카를로 등은 상대적으로 약함
- 그러나 이는 **기관 수준의 리소스가 필요한 영역**
- 개인 연구자로서는 장기 백테스트 + 실시간 검증으로 대응

**총평:** 
개인 투자자가 공개한 자료 중 **투명성·재현성·보수성 측면에서 국내 최고 수준**. 거래비용 과대 계산은 감점이 아닌 안전마진으로 평가해야 함.

### 4. 투명성 및 제품화 역량 (5.0/5.0)

#### 4-1. 24시간 라이브 - 리테일 퀀트 중 독보적

**실시간 포지션 공개:**
- 한국장(코스피)·미국장(나스닥) CBVR 포지션
- 누적 성과 상시 노출
- 리밸런싱 예정 여부
- 선물 운용 시 Target Beta 정보
- 실행 가이드(장중 분할 주문 등)

**의미:**
- 대부분의 리테일 퀀트는 "전략 개요 + 백테스트"에서 멈춤
- WeJump는 **실시간 검증 가능성** 제공
- 이는 **자신의 전략에 대한 확신**을 보여주는 강력한 신호

#### 4-2. 문서 저장소 + 시뮬레이터 + AI Q&A - 설명가능성 극대화

**통합된 학습·검증 경험 제공:**
- 프레임워크 분석 PDF
- 전략 논문(~30페이지, 학술 형식)
- 로직 설명서 (v2.3 상세 명세)
- 타 전략 비교 문서
- **인터랙티브 시뮬레이터**
- **AI 전략 Q&A 인터페이스**

**사용자 플로우:**
"읽기(문서) → 체험(시뮬레이터) → 실시간 관찰(라이브)" 

**평가:**
국내 리테일 퀀트 중 **투명성·커뮤니케이션·제품화 역량 최상위**. 블랙박스를 철저히 배제하고 설명가능성을 극대화한 접근.

## II. 리테일 퀀트 전문가 비교 분석

### 1. 개인 퀀트 투자자/교육자 vs WeJump

**대중화형 vs WeJump Labs**

| 비교 항목 | 대중화형 | WeJump Labs |
|----------|-----------------|-----------------|
| **접근 철학** | 팩터 기반 (가치·모멘텀 등) | 운용 구조 기반 (레짐·리스크·구조적 알파) |
| **목표** | 누구나 따라할 수 있는 투자 | 실전 운용 가능한 시스템 구축 |
| **타겟 독자** | 투자 초보~중급 | 중급~고급 (시스템 이해 필요) |
| **콘텐츠 형식** | 도서·유튜브·강의 | 논문·시뮬레이터·실시간 신호 |
| **전략 폭** | 여러 팩터 전략 나열 | 단일 코어 극한 고도화 |
| **실행 난이도** | 낮음 (6개월 리밸런싱) | 높음 (이벤트 기반, 선물 선택적) |
| **투명성** | 백테스트 결과 공유 | 실시간 포지션 24시간 공개 |

**WeJump의 독특한 포지션:**
- 전략 나열형 콘텐츠 생산자가 아님
- **자신이 실전 사용할 시스템**을 구조적 알파 관점에서 고도화
- 교육보다는 **동료 검증(Peer Review)** 가능성 제공

### 2. 퀀트 콘텐츠 채널 vs WeJump Labs

**일반 리테일 퀀트 특징:**
- "10가지 전략, 20가지 조합" 식의 폭 넓은 메뉴
- 월/분기 정기 리밸런싱
- MDD/샤프 단순 계산
- 플랫폼(퀀트킹/젠포트) 의존
- 블로그 결과 공유

**WeJump 차별점:**
- **CBVR 한 개 코어를 버전 관리(1.0→2.0→2.3)**
- **이벤트 기반 리밸런싱** (거래 최소화)
- **재귀적 리스크 제어** (메타 레벨)
- **자체 구현** (Pine Script + Python)
- **실시간 검증 가능** (24시간 라이브)

### 3. 백테스트 중심 vs 시스템 중심

| 차원 | 일반 리테일 퀀트 | WeJump Labs |

## III. 특성(Characteristics) 차이 분석

### 1. 철학적 성향: "생존(Survival) > 수익(Alpha)"

**명시적 영향:**
- Spitznagel (테일리스크/생존)
- Soros (Reflexivity/재귀성)

**실제 구현:**
- 재귀적 리스크 제어 (Equity Curve 자기참조)
- 이중 확인 헤지 메커니즘
- 3-Bucket 자본 구조 (생존성 우선)

**국내 리테일 퀀트와의 차이:**
- 일반: "어떤 종목을 살까?" (팩터 헌팅)
- WeJump: "어떻게 다룰까?" (운용 엔지니어링)

### 2. 구조적 알파 추구 - 개인의 기관형 사고

**구조적 알파란:**
- 단순한 "좋은 신호"를 넘어
- 상품 구조의 비효율성을 이용 (레버리지 ETF Decay)
- 자본 구조의 최적화 (3-Bucket)
- 실행 효율성 개선 (이벤트 기반)

**WeJump의 접근:**
- 레버리지 ETF → 선물 복제 (Decay 회피)
- 정기 리밸런싱 → 이벤트 트리거 (비용 절감)
- 단순 손절 → 레짐 기반 시스템적 대응

**이는 전략 나열이 아닌:**
- **실전 사용할 시스템의 효율성 극대화**
- 개인이지만 기관형 문제의식

### 3. 설명가능성 집착 - 블랙박스 거부

**일반적 접근:**
- "이 전략 수익률 좋습니다"
- 백테스트 차트 제시
- 로직은 간략히만

**WeJump 접근:**
- "이렇게 작동하며, 왜 그런지 설명합니다"
- 논문형 문서 (가설→방법→검증)
- Pine Script 코드 분석
- 시뮬레이터로 직접 체험
- 실시간 신호로 검증 가능

**의미:**
- 자신의 전략에 대한 확신
- 동료 검증 (Peer Review) 환영
- 피드백 수용 자세

### 4. 버전 관리 - 전략을 제품처럼

**CBVR 진화:**
- 1.0: 기본 3중 필터
- 2.0: VXX 이중 확인 헤지 추가
- 2.3: 선물 복제 + 3-Bucket + PFIX 동기화

**이는:**
- 단발성 전략이 아닌 **플랫폼/프레임워크** 취급
- 지속적 개선과 진화
- 명확한 버전별 차별점 문서화
